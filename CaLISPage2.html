In questa pagina sono presentati vari frammenti di codice al fine di poterne valutare l'utilità. Sono da considerare come strumenti a se stanti in vista di una loro eventuale integrazione nel progetto, anche solo per ricordare ciò che è possibile fare.
Il nome di ogni frammento è ordinabile (prime 4 cifre numeriche) e cerca di essere mnemonico, con una descrizione minimale delle principali caratteristiche.
In generale il formato di ogni frammento prevede un titolo al di sotto del quale può esserci una immagine del risultato e un link da cliccare per eseguire il frammento stesso, di fianco c'è riquadro con una descrizione.
Si fa notare che il frammento viene eseguito in una nuova pagina. Per tornare alla pagina presente basta cliccare la freccia di ritorno della nuova pagina aperta (sul PC in alto a sinistra, in un dispositivo Android in basso a destra).

(testo provvisorio)
Il link apre una pagina web contenente una "scheda" dedicata ai vari modi di rappresentare un carattere
con i suoi eventuali diacritici rappresentanti un (fonema?)
La particolarità di questa scheda consiste nell'essere alimentata da un file JSON prodotto dalla estrazione in banca dati
delle informazioni necessarie, secondo un determinato schema(in questo esempio il file è emulato assegnando i suoi valori
direttamente all'interno dello script)
E' preso in esame un caso semplice (la "e" con accento grave) che però già presenta la particolarità di essere rappresentato
con due caratteri distinti del font ALI. Viene anche indicata la stringa in codice di inserimento ALI per rappresentare i
caratteri con il font ALI. I glifi vengono anche disegnati in modo vettoriale
Dal punto di vista tecnico il codice Html "legge" (in queto caso virtualmente) i dati JSON e contiene uno script javascript
il quale produce degli elementi SVG (testi e vettori)
Detti che elementi SVG rappresentano sia testi selezionabili (quindi copiabili), sia disegni vettoriali non selezionabili
Le informazioni sono inserite secondo uno schema logico di righe che prevede due diverse tipologie di rappresentazione:
  -un codice carattere e la sua descrizione
  -un disegno vettoriale e il suo significato nel proprio sistema di codifica
Quello che si vorrebbe realizzare è un sistema di visualizzazione automatizzata di supporto alla comparazione dei diversi
modelli di simbolizzazione (fonetica?).
In questa versione di test lo script prevede un numero massimo di 13 righe nella "scheda" (che tratta di un solo "argomento")

Questo link esegue un file html il quale:
- esegue uno script di tipo JavaScript utilizzando la libreria p5.min.js
- scrive un testo html contenente un ideogramma “﨩” (U+FA29) sia copiandolo direttamente
  nella stringa, sia rappresentandolo tramite il suo codice UTF-8
Lo script invece:
- legge un file di dati memorizzati nel formato di scambio JSON contenente un oggetto
  composto di 2 record "linguistici".
- disegna sia la localizzazione di questi due record che il contenuto dei campi di cui sono formati
  (quando si sovrappone il cursore sopra i cerchi che lo rappresentano e che hanno un diametro
  relativo al campo 'locutori stimati')

Al contrario del testo html, la stringa contenuta nel file JSON riproduce l'ideogramma solo copiato
ma non rappresentato tramite il suo codice UTF-8

Ad ogni buon conto tutti i caratteri sono rappresentate "lungo la stessa y" della stringa di appartenenza.
In un prossimo frammento si proverà a comporre una stringa "multilivello" tramite il formato SVG
(Scalable Vector Graphics) integrabile nel formato html stesso
